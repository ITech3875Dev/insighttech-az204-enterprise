# Module M01: Blob Storage Application Workflows

## Module Overview
| Item | Details |
|------|---------|
| Duration | 2.5 to 3.5 hours |
| Difficulty | Beginner to Intermediate |
| Primary services | Azure Storage Account, Blob Storage |
| Labs | Beginner and Advanced linkage |

## Learning Objectives
- Build application workflows that upload, list, download, and validate blobs.
- Apply secure storage account defaults and container design choices.
- Evaluate lifecycle and access decisions for storage-backed applications.

## Module Flow
1. Create secure storage account baseline.
2. Implement SDK-based upload and retrieval flow.
3. Validate content, metadata, and container behavior.
4. Apply lifecycle and security hardening in later stages.

## Required Deliverables
- Blob upload and retrieval output.
- Container and storage account configuration evidence.
- Design notes for access and lifecycle choices.
- Validation output from LP02 script.

## Associated Labs
- `labs/beginner/lab-01-blob-sdk-workflows/README.md`
- `labs/advanced/lab-01-storage-security-and-lifecycle-hardening/README.md`

## Exit Criteria
- Learner can implement and verify a Blob SDK workflow.
- Learner can explain secure storage defaults and lifecycle decisions.
- Learner can validate storage behavior with CLI and PowerShell evidence.
