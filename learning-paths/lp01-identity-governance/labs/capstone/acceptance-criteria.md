# Capstone Lab: Acceptance Criteria\n\n## Pass/Fail Criteria\n\nAll of the following must be true for the capstone lab submission to be accepted:\n\n### Structural Requirements\n- [ ] All files are in `learning-paths/lp01-identity-governance/labs/capstone/` directory\n- [ ] No hardcoded secrets (keys, tokens, passwords) in any files\n- [ ] All markdown files use proper GitHub markup (headers, lists, links)\n- [ ] All JSON/PowerShell files are valid (parseable without errors)\n\n### Part A: RBAC Architecture (Must pass)\n- [ ] `diagrams/rbac-architecture.md` exists and contains clear RBAC hierarchy diagram\n- [ ] `evidence/rbac-assignments.json` contains valid JSON from `az role assignment list`\n- [ ] `rationale/rbac-design-rationale.md` explains all 6 role assignments and WHY (least-privilege principle)\n- [ ] Validation script confirms all role assignments exist (PASS for RBAC section)\n\n### Part B: Policy & Compliance (Must pass)\n- [ ] `policies/tag-policy-definitions.json` contains valid policy JSON (tag audit + deny)\n- [ ] `policies/location-policy.json` contains valid deny policy\n- [ ] `policies/storage-policy.json` contains valid storage policy (audit or deny)\n- [ ] `evidence/policy-assignments.txt` shows all 3 policies assigned (via `az policy assignment list`)\n- [ ] `evidence/compliance-report.json` shows non-compliant resources BEFORE remediation  \n- [ ] `evidence/compliance-remediation-log.txt` documents at least 1 full remediation workflow\n- [ ] `rationale/policy-exemption-justification.md` justifies all exemptions\n- [ ] Validation script confirms all policies are assigned and not disabled (PASS for Policy section)\n\n### Part C: Cost Management (Must pass)\n- [ ] `evidence/budgets-created.txt` shows 2+ budgets (Infra ~$500, AppDev ~$300) via `az costmanagement budget list`\n- [ ] `evidence/cost-analysis-by-costcenter-tag.csv` shows costs grouped by CostCenter tag\n- [ ] `rationale/advisor-recommendations.md` lists 3 recommendations, categorizes impact/cost, and states Accept/Defer/Reject decision\n- [ ] Validation script confirms budget query succeeds (PASS for Cost section)\n\n### Part D: Break-Glass & Incident Response (Should pass for 70%+, optional for minimum)\n- [x] `runbooks/break-glass-access-procedure.md` is well-documented and clear\n- [ ] `evidence/break-glass-activity-log.txt` shows at least 1 test event with:\n  - [ ] Role assignment creation (with timestamp)\n  - [ ] Actions during break-glass (e.g., resource deletion)\n  - [ ] Role assignment removal or auto-revoke\n- [ ] Runbook includes: Goal → Prerequisites → Request flow → Grant → Usage → Revoke → Audit\n- [ ] Validation script confirms activity log entries exist (PASS for Break-Glass section)\n\n### Part E: Validation & Self-Assessment (Must pass)\n- [ ] `validation/validate-capstone.ps1` exists and runs without errors\n- [ ] Validation script performs all 5 checks (RBAC, Policies, Budgets, Break-Glass, Compliance)\n- [ ] Validation script outputs clear PASS/FAIL for each section (or SKIP if not required)\n- [ ] `evidence/capstone-completion-checklist.md` is fully completed (all boxes checked if items exist)\n- [ ] `reflection/learnings.md` is 300—500 words and answers all 4 reflection questions\n\n---\n\n## Grading Outcome Matrix\n\n| Score | Status |
