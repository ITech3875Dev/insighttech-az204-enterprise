# M02 - VM Scale and Availability

## Module Overview
| Item | Details |
|------|----------|
| **Duration** | Beginner: ~55 min \| Intermediate: ~70 min \| Advanced: ~95 min |
| **Prerequisites** | M01 completion recommended |
| **Difficulty** | All tiers available |
| **Labs** | Beginner + Intermediate + Advanced alignment |

## Learning Outcomes
You will:
- Deploy and configure Virtual Machine Scale Sets (VMSS)
- Configure autoscale rules and capacity boundaries
- Apply availability zones/sets and load-balancing patterns
- Validate scale and resilience behavior with command evidence

See [objectives.md](objectives.md) for AZ-104 mapping.

## Prerequisites & Permissions
- Non-production lab subscription
- Contributor role at target scope
- Azure Portal, Azure CLI, and PowerShell Az installed

## Accessibility & Support
- Equivalent portal and command workflows are provided
- Compliance evidence can be generated in JSON and CSV formats

## Self-Assessment Path
1. Verify VMSS and availability prerequisites
2. Complete labs in sequence
3. Run validation script
4. Document findings and remediation options

## Deliverables (Definition of Done)
- VMSS scaling and availability configuration evidence captured
- Scale behavior tests include expected increase/decrease scenarios
- Validation script returns `PASS`
- References are official Microsoft Learn/Docs
